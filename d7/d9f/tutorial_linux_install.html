下一个教程: Using OpenCV with gcc and CMake

原作者Ana Huamán
兼容性OpenCV >= 3.0

快速启动

构建核心模块

# Install minimal prerequisites (Ubuntu 18.04 as reference)
sudo apt update && sudo apt install -y cmake g++ wget unzip
# Download and unpack sources
wget -O opencv.zip https://github.com/opencv/opencv/archive/master.zip
unzip opencv.zip
# Create build directory
mkdir -p build && cd build
# Configure
cmake ../opencv-master
# Build
cmake --build .

构建opencv_contrib

# Install minimal prerequisites (Ubuntu 18.04 as reference)
sudo apt update && sudo apt install -y cmake g++ wget unzip
# Download and unpack sources
wget -O opencv.zip https://github.com/opencv/opencv/archive/master.zip
wget -O opencv_contrib.zip https://github.com/opencv/opencv_contrib/archive/master.zip
unzip opencv.zip
unzip opencv_contrib.zip
# Create build directory and switch into it
mkdir -p build && cd build
# Configure
cmake -DOPENCV_EXTRA_MODULES_PATH=../opencv_contrib-master/modules ../opencv-master
# Build
cmake --build .

详细流程

本节提供了构建过程的更多细节,并描述了替代方法和工具.请参阅OpenCV installation overview教程的一般安装细节和OpenCV configuration options reference配置选项文档.

安装编译器和生成工具

  • 要编译OpenCV,你需要一个C++编译器.通常是G++/GCC或Clang/LLVM:
    • Install GCC...
      sudo apt install -y g++
    • ... or Clang:
      sudo apt install -y clang
  • OpenCV使用CMake构建配置工具:
    sudo apt install -y cmake
  • CMake可以为不同的构建系统生成脚本,例如.make,ninja:
    • 安装Make...
      sudo apt install -y make
    • ... 或 Ninja:
      sudo apt install -y ninja-build
  • 安装获取和解包源的工具:
    • wget和unzip...
      sudo apt install -y wget unzip
    • ... 或git:
      sudo apt install -y git

下载源

获取OpenCV源代码有两种方法:

  • 使用web浏览器或任何下载工具（约80-90Mb）下载存储库的快照并将其解压缩...
    wget -O opencv.zip https://github.com/opencv/opencv/archive/master.zip
    unzip opencv.zip
    mv opencv-master opencv
  • ... 或使用将存储库克隆到本地计算机git获取完整的更改历史记录 (>470Mb):
    git clone https://github.com/opencv/opencv.git
    git -C opencv checkout master
Note
其他分支、版本或提交的快照可以在GitHub以及官方下载页.

配置和生成

  • 创建生成目录:
    mkdir -p build && cd build
  • 配置-为首选生成系统生成生成脚本:
    • 为了make...
      cmake ../opencv
    • ... 或为ninja:
      cmake -GNinja ../opencv
  • 生成-运行实际编译过程:
    • Usingmake...
      make -j4
    • ... 或ninja:
      ninja
Note
Configure(配置)进程可以从internet下载一些文件以满足库依赖关系,连接失败可能导致某些模块或功能关闭或行为不同.参考OpenCV installation overview和OpenCV configuration options reference有关详细信息和完整配置选项参考的教程.
如果在生成过程中遇到问题,请尝试清理或重新创建生成目录.配置中的更改（如禁用依赖项、修改构建脚本或将源切换到另一个分支）处理得不是很好,可能会导致工作区中断.
Make可以并行运行多个编译进程,-j<NUM>选项表示 "运行 <数量> 工作同时进行".Ninja将自动检测可用处理器内核的数量,而不需要-j选项.

检查生成结果

成功构建后,您将在build/lib目录和可执行文件（测试、示例、应用程序）build/bin目录:

ls bin
ls lib

CMake包文件将位于生成根目录中:

ls OpenCVConfig*.cmake
ls OpenCVModules.cmake

安装

警告
安装过程只会将文件复制到预定义的位置并进行小修补.使用此方法安装的库未集成到系统包注册表中,无法自动卸载.由于可能与系统包冲突,我们不建议常规用户在系统范围内安装.

默认情况下,OpenCV将安装到/usr/local目录中,所有文件将复制到以下位置:

  • /usr/local/bin- executable files
  • /usr/local/lib- libraries (.so)
  • /usr/local/cmake/opencv4- cmake package
  • /usr/local/include/opencv4- headers
  • /usr/local/share/opencv4-其他文件（如XML格式的训练级联）

Since/usr/local属于根用户,安装应以提升的权限执行(sudo):

sudo make install

或

sudo ninja install

安装根目录可以用CMAKE_INSTALL_PREFIX配置参数,例如.-DCMAKE_INSTALL_PREFIX=$HOME/.local安装到当前用户的本地目录.安装布局可以通过OPENCV_*_INSTALL_PATH参数.看到了吗OpenCV configuration options reference详情.
