在您的计算机上安装OpenCV有两种方法:下载平台的预构建版本或从源代码处编译.

Prebuilt version(预构建版本)

在许多情况下,您可以找到满足您需求的预构建版本的OpenCV.

OpenCV核心团队的软件包

Android、iOS和Windows的软件包使用默认参数和最新的编译器构建,每个版本都会发布,它们不包含opencv_contrib modules.

  • GitHub releases:https://github.com/opencv/opencv/releases
  • SourceForge.net:https://sourceforge.net/projects/opencvlibrary/files/

Third-party packages(第三方软件包)

其他组织和人员维护自己的OpenCV二进制发行版.例如:

  • 流行Linux发行版中的系统包(https://pkgs.org/search/?q=opencv)
  • PyPI (https://pypi.org/search/?q=opencv)
  • Conda (https://anaconda.org/search?q=opencv)
  • Conan (https://github.com/conan-community/conan-opencv)
  • vcpkg (https://github.com/microsoft/vcpkg/tree/master/ports/opencv)
  • NuGet (https://www.nuget.org/packages?q=opencv)
  • Brew (https://formulae.brew.sh/formula/opencv)
  • Maven (https://search.maven.org/search?q=opencv)

Build from sources(从源代码生成)

可能现有的二进制软件包不适用于您的用例,那么您必须自己构建OpenCV的定制版本.本节提供了构建过程的高级概述,请参阅教程以了解具体平台的实际构建说明.

OpenCV usesCMake构建配置和构建的管理系统,因此本节主要描述用CMake构建软件的一般过程.

步骤0:先决条件

安装C++编译器和构建工具.在*NIX平台上,它通常是GCC/G++或Clang编译器和Make或Ninja构建工具.在Windows上,它可以是visualstudioide或MinGW-w64编译器.Android NDK中提供了Android的原生工具链.XCode IDE用于为OSX和iOS平台构建软件.

从官方网站或其他来源安装CMake.

获取其他第三方依赖项:具有解码视频或显示GUI元素等额外功能的库；提供选定算法的优化实现的库；用于生成文档和其他附加功能的工具.检查OpenCV configuration options reference获取可用选项和相应的依赖项.

步骤1:获取软件源

典型的软件项目由一个或多个代码库组成. OpenCV有两个代码库:opencv具有稳定且积极支持的算法和 opencv_contrib 包含实验性和非自由(专利)算法；以及一个包含测试数据的存储库: opencv_extra .

您可以下载存储库的快照,其形式为存档或具有完整历史记录的克隆存储库.

要下载快照存档,请执行以下操作:

  • Go tohttps://github.com/opencv/opencv/releases并从任何版本下载“源代码”存档.
  • (可选)转到https://github.com/opencv/opencv_contrib/releases并下载“源代码”存档,其版本与 opencv
  • (可选)转到https://github.com/opencv/opencv_extra/releases并下载“源代码”存档,其版本与 opencv
  • 把所有的档案打包到某个地方

要克隆存储库,请在console中运行以下命令(git must be installed(git 必须安装)):

git clone https://github.com/opencv/opencv
git -C opencv checkout <some-tag>
# optionally
git clone https://github.com/opencv/opencv_contrib
git -C opencv_contrib checkout <same-tag-as-opencv>
# optionally
git clone https://github.com/opencv/opencv_extra
git -C opencv_extra checkout <same-tag-as-opencv>
Note(注意)
如果要使用多个存储库构建软件,请确保所有组件彼此兼容.对于OpenCV,这意味着opencv和opencv_contrib必须在同一标记处签出存储库,或者从同一版本下载所有快照存档.
在选择要下载的版本时,请考虑您的目标平台和开发工具版本,最新版本的OpenCV可能会在非常旧的编译器上出现构建问题,反之亦然.我们建议使用最新版本和最新的OS/编译器组合.

第2步:配置

在这一步,CMake将验证所有必要的工具和依赖项是否可用并与库兼容,并为所选的构建系统生成中间文件.它可以是makefile、IDE项目和解决方案等.通常在新创建的构建目录中执行此步骤:

cmake -G<generator> <configuration-options> <source-directory>
Note(注意)
cmake-gui应用程序允许使用图形用户界面查看和修改可用选项.看到了吗https://cmake.org/runningcmake/详情.

步骤3:构建

在构建过程中,源文件被编译成对象文件,这些文件被链接在一起或以其他方式组合到库和应用程序中.可以使用通用命令运行此步骤:

cmake --build <build-directory> <build-options>

... 或者可以直接调用底层生成系统:

make

步骤3:安装

在安装过程中,生成目录中的生成结果和其他文件将复制到安装位置.默认安装位置为/usr/local在UNIX和C:/Program Files在窗户上.在配置步骤中,可以通过设置CMAKE_INSTALL_PREFIX选项.要执行安装,请运行以下命令:

cmake --build <build-directory> --target install <other-options>
注意
这个步骤是可选的,OpenCV可以直接从build目录中使用.
如果安装根目录是受保护的系统目录,则安装过程必须以超级用户或管理员权限(例如.sudo cmake ...).
