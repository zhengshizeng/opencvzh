图像监视:在 Visual Studio 调试器中查看内存中的图像

Image Watch是Microsoft Visual Studio的一个插件，它允许您可视化内存中的图像(cv::Mat或IplImage_对象)，而调试应用程序。这有助于跟踪bug，或者简单地理解给定的代码段在做什么。

先决条件

本教程假设您具有以下可用功能:

  1. 安装了更新1的Visual Studio 2012 Professional(或更高版本)。可以下载更新1在这里.
  2. 在Windows计算机上安装OpenCV(教程:Installation in Windows).
  3. 能够在Visual Studio中创建和构建OpenCV项目(教程:How to build applications with OpenCV inside the "Microsoft Visual Studio").

安装

下载图像监视安装程序(Visual Studio 2019 | Visual Studio 2017 | Visual Studio 2012、2013、2015)安装程序包含一个扩展名为.vsix的文件(Visual Studio 扩展). 要启动它，只需在Windows资源管理器中双击.vsix文件。安装程序完成后，请确保重新启动Visual Studio以完成安装。

Example(例子)

Image Watch适用于任何使用OpenCV图像对象的现有项目(例如，cv::Mat). 在这个例子中，我们使用一个最小的测试程序，从一个文件中加载一个图像并运行一个边缘检测器。要构建该程序，请在visualstudio中创建一个控制台应用程序项目，将其命名为“imagewatchdemo”，并在下面插入源代码。

// Test application for the Visual Studio Image Watch Debugger extension
#include <iostream> // std::cout
#include <opencv2/core/core.hpp> // cv::Mat
#include <opencv2/imgcodecs/imgcodecs.hpp> // cv::imread()
#include <opencv2/imgproc/imgproc.hpp> // cv::Canny()
using namespace std;
using namespace cv;
void help()
{
cout
<< "----------------------------------------------------" << endl
<< "This is a test program for the Image Watch Debugger " << endl
<< "plug-in for Visual Studio. The program loads an " << endl
<< "image from a file and runs the Canny edge detector. " << endl
<< "No output is displayed or written to disk."
<< endl
<< "Usage:" << endl
<< "image-watch-demo inputimage" << endl
<< "----------------------------------------------------" << endl
<< endl;
}
int main(int argc, char *argv[])
{
help();
if (argc != 2)
{
cout << "Wrong number of parameters" << endl;
return -1;
}
cout << "Loading input image: " << argv[1] << endl;
Mat input;
input = imread(argv[1], IMREAD_COLOR);
cout << "Detecting edges in input image" << endl;
Mat edges;
Canny(input, edges, 10, 100);
return 0;
}

确保您的活动解决方案配置(Build–>配置管理器)设置为调试版本(通常称为“调试”)。这将禁用编译器优化，以便在调试器中查看变量可以可靠地工作。

构建您的解决方案(Build–>生成解决方案，或按F7).

在继续之前，不要忘记将输入图像的命令行参数添加到项目中(右键单击project–>属性–>配置属性–>调试，然后使用映像的位置设置字段命令参数)。

现在在源代码行上设置一个断点

Mat edges;

要设置断点，请右键单击源代码行并选择断点–>从上下文菜单中插入断点。

在调试器中启动程序(Debug–>开始调试，或点击F5). 当到达断点时，程序将暂停，Visual Studio将在断点处显示黄色指令指针:

现在可以检查程序的状态。例如，你可以把局部变量窗口(调试–>窗口–>局部变量)，它将显示当前作用域中变量的名称和值:

请注意，内置局部变量窗口将只显示文本。这就是图像手表插件的用武之地。图像手表就像另一个局部变量窗口，但内置了图像查看器。要打开图像监视，请选择查看–>其他窗口–>图像监视。像visualstudio的局部变量窗口，图像观看可以停靠到Visual Studio IDE。另外，VisualStudio还将记住是否打开了图像监视，以及它在调试会话之间的位置。这意味着您只需执行一次;下次开始调试时，“图像监视”将回到原来的位置。以下是停靠图像监视窗口在断点处的外观:

左上角的单选按钮(本地人/观察)选择显示在图像列表下图:局部变量列出当前范围内的所有OpenCV图像对象(此列表将自动填充)。观察显示已固定以进行连续检查的图像表达式(此处未描述，请参阅)图像监视文档详细信息)。图像列表显示基本信息，如宽度、高度、通道数，以及缩略图(如果可用)。在我们的示例中，图像列表包含两个局部图像变量，输入和边缘.

如果图像有缩略图，左键单击该图像将在图像查看器在右边。查看器允许您平移(拖动鼠标)和缩放(鼠标滚轮)。它还显示当前鼠标位置的像素坐标和值。

注意列表中的第二个图像，边缘，显示为“无效”。这表示此图像对象的某些数据成员的值已损坏或无效(例如，负图像宽度)。这是程序中的这一点，因为C++构造函数边缘尚未运行，因此其成员具有未定义的值(在调试模式下，它们通常用“0xCD”字节填充)。

从这里您可以单步完成代码(调试->；跨过，或按F10)观察像素的变化:如果你一步一个脚印Mat edges;声明边缘图像将从“invalid”变为“empty”，这意味着它现在处于有效状态(默认构造)，即使它尚未初始化(使用cv::Mat::create()，例如)。如果你再跨一步cv::Canny()调用时，您将看到边缘图像的缩略图出现在图像列表中。

现在假设您想对cv::Canny()实施。带上边缘通过在图像列表并放大具有清晰边界的区域:

右键单击图像查看器打开视图上下文菜单并启用链接视图(菜单项旁边的复选框指示是否启用该选项)。

在相同大小的图像之间翻转时，“链接视图”功能使视图区域保持固定。要了解其工作原理，请从图像列表中选择输入图像;现在，您应该可以在输入图像中看到相应的放大区域:

也可以使用上/下光标键在查看输入和边之间来回切换。这样您就可以很容易地验证检测到的边缘是否与输入图像中的数据很好地对齐。

More(更多)...

图像手表有许多更先进的功能，如

  1. 将图像固定到Watch用于跨作用域或调试会话之间检查的列表
  2. 直接在监视窗口内钳制、阈值化或区分图像
  3. 将内存中的图像与文件中的参考图像进行比较
